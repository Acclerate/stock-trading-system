# 风险管理技能

负责交易风险控制和管理，包括止损止盈、仓位控制、风险预警。

## 使用方式

```
/risk-manager                           # 检查整体风险
/risk-manager --code 002202             # 检查单只股票风险
/risk-manager --warning                 # 检查预警状态
/risk-manager --portfolio               # 显示投资组合概览
```

## 止损系统

### 止损类型

**1. 技术止损**
```
触发条件：
□ 跌破关键支撑位
□ MACD死叉且放量
□ 跌破上升趋势线
□ 形态破位

执行：立即市价卖出
```

**2. 金额止损**
```
触发条件：
☢ 亏损达到10%（强制）
☢ 当日亏损>5%（警告）
☢ 当周亏损>15%（强制）

执行：立即市价卖出
```

**3. 时间止损**
```
触发条件：
□ 持仓>3个月未达目标
□ 长期横盘无起色

执行：重新评估，考虑卖出
```

### 止损执行流程

```
止损触发 → 立即通知用户 → 用户确认 → 执行卖出 → 复盘总结
```

### 止损纪律

```
【铁律】
  ☢ 触发止损必须执行
  ☢ 不可以有侥幸心理
  ☢ 不可以临时修改止损价
  ☢ 不可以等待反弹再卖

【禁止】
  ✗ "再等等可能会反弹"
  ✗ "已经跌这么多了，等等看"
  ✗ "补仓摊低成本"
  ✗ "先卖一部分看看"

【后果】
  不执行止损的后果：
    - 小亏变大亏
    - 大亏变爆仓
    - 失去本金，无法翻身
```

## 止盈系统

### 止盈类型

**1. 金额止盈**
```
目标：盈利20%
执行：至少卖出50%
```

**2. 技术止盈**
```
触发条件：
□ 远离均线>10%
□ 连续大涨后滞涨
□ 放量滞涨或高位震荡
□ 顶背离信号

执行：分批卖出
```

**3. 时间止盈**
```
触发条件：
□ 持仓达到预期时间
□ 达到目标价位

执行：重新评估，决定是否继续持有
```

### 止盈执行策略

```
盈利0-10%：
  → 继续持有
  → 严格执行止损

盈利10-20%：
  → 观察技术面
  □ 技术面强：继续持有
  □ 技术面弱：部分止盈

盈利20-30%：
  → 至少卖出50%
  → 剩余设置移动止损

盈利30%以上：
  → 分批止盈
  → 保留底仓

盈利达到目标：
  → 全部或部分止盈
  → 落袋为安
```

### 移动止损

```
定义：保护已实现的利润，同时保留上涨空间

执行方式：
  盈利10%后：
    → 止损调整为成本价

  盈利20%后：
    → 止损调整为+10%位置

  盈利30%后：
    → 止损调整为+20%位置

  盈利50%后：
    → 止损调整为+30%位置

示例：
  买入价：25元
  现价：30元（盈利20%）
  移动止损：25 × 1.1 = 27.5元

  如果股价跌到27.5元：止损，保住部分利润
  如果股价继续上涨：继续持有，享受更多利润
```

## 核心原则

1. **本金第一**：保护本金是最高优先级
2. **止损必行**：触发止损必须执行
3. **不摊不平**：亏损后绝不加仓摊平
4. **仓位控制**：永远不满仓一只股票
5. **风险可控**：每笔交易风险必须可控
