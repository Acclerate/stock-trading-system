## 一、交易系统架构

### 1.1 核心原则

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  交易铁律（不可违反）                                        │
│                                                             │
│  1. 不做短线 → 持股周期≥1个月                                │
│  2. 不重仓 → 单只股票≤20%                                   │
│  3. 不追高 → 不买涨停板、不买连续大涨                        │
│  4. 必止损 → 亏损10%触发                                    │
│  5. 不摊平 → 亏损后不加仓                                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 决策流程

```
交易前评估 → 技术分析 → 风险评估 → 仓位计算 → 执行交易 → 复盘总结
    ↓          ↓          ↓          ↓          ↓          ↓
  检查清单   多维确认   风险收益比   资金管理   严格执行   吸取教训
```

## 三、交易决策矩阵

```
是否满足所有条件？┬─ YES ─┬→ 技术面评分≥7分？
                  │       │

                  │       │        │
                  │       │        ├─ YES ─┬→ 仓位≤20%？
                  │       │        │        │
                  │       │        │        ├─ YES → 可以买入
                  │       │        │        └─ NO → 减少仓位
                  │       │        └─ NO → 放弃
                  │       └─ NO → 放弃
                  └─ NO → 放弃

买入后检查清单：
  □ 是否设止损？是（10%）
  □ 是否设止盈？是（20%）
  □ 仓位是否≤20%？是
  □ 是否不追高？是
  □ 是否有明确计划？是
```

### 3.2 卖出决策树

```
触发条件（任一满足即卖出）：

□ 止损触发
  └─ 亏损≥10% → 立即卖出

□ 止盈触发
  └─ 盈利≥20% → 分批卖出

□ 技术面转坏
  ├─ 跌破关键支撑
  ├─ MACD死叉
```

## 四、风险控制系统

### 4.1 三个层次的风控

```
第一层：事前风控
  ├─ 单只股票仓位≤20%
  ├─ 总仓位≤80%
  ├─ 必须设止损
  └─ 不追高不摊平

第二层：事中风控
  ├─ 实时监控止损
  ├─ 触发立即执行
  ├─ 不心存侥幸
  └─ 不临时改变计划

第三层：事后风控
  ├─ 复盘每笔交易
  ├─ 总结经验教训
  ├─ 调整策略
  └─ 持续优化
```

### 4.2 仓位管理矩阵

```
              技术面强    技术面弱    技术面中性
                │          │          │
基本面强    │  30%      │  10%      │  20%
────────────┼──────────┼──────────┼──────────
基本面弱    │  禁止      │  禁止      │  禁止
────────────┼──────────┼──────────┼──────────
基本面中性  │  20%      │  观望      │  10%
```

### 4.3 止损止盈标准

```
止损类型：

1. 技术止损
   - 跌破关键支撑位
   - MACD死叉
   - 放量下跌
   - 执行：立即

2. 金额止损
   - 亏损10%
   - 执行：立即

3. 时间止损
   - 持有3个月未达目标
   - 执行：重新评估

止盈类型：

1. 技术止盈
   - 连续大涨
   - 远离均线
   - 放量滞涨
   - 执行：分批

2. 金额止盈
   - 盈利20%
   - 执行：至少卖出50%

3. 目标止盈
   - 达到目标价
   - 执行：至少卖出50%
```

## 七、异常情况应对

### 7.1 突发利空

```
应对流程：
1. 立即评估影响程度
   - □ 重大利空（-10%以上）
   - □ 一般利空（-5%~-10%）
   - □ 轻微利空（-5%以内）

2. 决策
   - 重大利空 → 立即清仓
   - 一般利空 → 减仓50%
   - 轻微利空 → 观望

3. 执行
   - 开盘即挂单
   - 不要犹豫
   - 不要幻想
```

### 7.2 突发利好

```
应对流程：
1. 评估利好真实性
   - □ 确定性利好
   - □ 传闻性利好

2. 决策
   - 确定性利好 → 持有或加仓
   - 传闻性利好 → 观望

3. 执行
   - 不要追高
   - 等回调确认
   - 分批建仓
```

### 7.3 大跌应对

```
大盘单日跌幅>3%：

□ 检查个股跌幅
  - 个股跌幅<大盘：可能强势
  - 个股跌幅>大盘：可能弱势

□ 评估持仓
  - 亏损是否触及止损？
  - 基本面是否恶化？
  - 技术面是否破位？

□ 决策
  - 触发止损 → 立即执行
  - 未触发止损 → 观望
  - 基本面恶化 → 立即清仓
```

### 7.4 大涨应对

```
大盘单日涨幅>3%：

□ 检查个股涨幅
  - 个股涨幅>大盘：强势
  - 个股涨幅<大盘：弱势

□ 评估持仓
  - 是否达到止盈？
  - 是否量价背离？
  - 是否滞涨？

□ 决策
  - 达到止盈 → 分批卖出
  - 滞涨 → 考虑换股
  - 继续强势 → 持有
```
