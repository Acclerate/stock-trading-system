# stockRanking 配置文件说明

## 配置文件位置
`config/stock_ranking.json`

## 配置文件格式

```json
{
    "stocks": [
        {
            "code": "600489",
            "name": "中金黄金"
        },
        {
            "code": "601088",
            "name": "中国神华"
        }
    ],
    "description": "多维评分分析股票池",
    "update_time": "2026-03-06"
}
```

## 配置项说明

### stocks（必需）
- **类型**: 对象数组
- **说明**: 要分析的股票列表，每个股票包含代码和名称
- **格式**: 
  - `code`: 6位数字代码（不带市场后缀）
  - `name`: 股票中文名称（仅用于配置文件阅读，不影响程序运行）
- **示例**: 
  ```json
  {
      "code": "600489",
      "name": "中金黄金"
  }
  ```

### description（可选）
- **类型**: 字符串
- **说明**: 股票池描述
- **示例**: "多维评分分析股票池"

### update_time（可选）
- **类型**: 字符串
- **说明**: 最后更新时间
- **格式**: YYYY-MM-DD
- **示例**: "2026-03-06"

## 注意事项

1. **股票代码格式**: 使用6位数字代码，不带 `.SZ` 或 `.SH` 后缀
2. **股票名称格式**: 中文名称仅用于配置文件阅读，程序只读取 `code` 字段
3. **代码有效性**: 确保股票代码在A股市场存在
4. **数量建议**: 建议5-20只股票，过多会影响执行速度
5. **更新时间**: 每次修改股票列表后，建议更新 `update_time` 字段
6. **备份建议**: 修改前建议备份原配置文件

## 错误处理

如果配置文件不存在或格式错误，程序会：
1. 输出警告信息
2. 使用默认的15只股票列表
3. 继续执行分析

## 配置文件优先级

1. **配置文件存在**: 使用配置文件中的股票列表（只读取 `code` 字段）
2. **配置文件不存在**: 使用默认的15只股票
3. **配置文件格式错误**: 使用默认的15只股票

## 修改配置文件后的操作

修改配置文件后，直接运行 `stockRanking.py` 即可，无需重新编译或安装：

```bash
python strategies/stockRanking.py
```

程序会自动读取新的配置文件并使用新的股票列表。

## 程序读取逻辑说明

程序从配置文件读取时：
1. 读取 `stocks` 数组
2. 提取每个股票对象的 `code` 字段
3. 忽略 `name` 字段（仅用于配置文件阅读）
4. 使用提取的代码列表进行分析

示例代码：
```python
stocks = config.get('stocks', [])
symbols = [stock['code'] for stock in stocks]  # 只提取代码
```
